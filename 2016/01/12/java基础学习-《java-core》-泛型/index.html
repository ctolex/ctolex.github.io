java泛型基础学习

定义简单泛型类

  • 泛型可以有多个类型变量 public class Pair<T, U> { ... }。
  • 元素类型常用大写形式表示，Java库中使用变量 E 表示集合中的元素类型，K 和 V 分别表示表中的关键字和值的类型，T表示任何类型。

类型变量的限定

  • 表示T应该是绑定类型（Comparable）的子类型，注意使用的是extends关键字，不是implements。
  • T和绑定类型可以是类，也可以是接口。
  • 一个类型变量可以由多个限定 T extends Comparable & Serializable，限定类型用 & 分隔，而类型变量用 , 分隔。
  • 注意：限定中至多有个一个类，可以有多个接口。如果用一个类作为限定，它必须是限定列表中的第一个。

泛型代码和虚拟机

  • 虚拟机没有泛型类型，所有对象都属于普通类。
  • 无论何时定义的一个泛型类，都自动提供了一个相应的原始类型。擦除类型变量，并替换为限定类型（无限定类型的变量用Object）
  • 注意：原始类型用第一个限定的变量来替换，如果没有给定限定就用Object替换。如果有多个限定类型，应该将标签（tagging）接口（没有方法的接口）放在边界列表的末尾。
  • 调用泛型方法时，如果擦除返回类型，编译器将强制插入类型装换。
  • 桥方法被合成来保持多态。

  • 关闭对方法的检查,必须放在生成警告的代码之前，或者标注整个方法（标注会关闭对方法中的所有方法的检查）

泛型约束与局限性

  • 不能用基本类型实例化类型参数。没有 Pair<double>，只有 Pair<Double>，原因是Pair类含有Object类型的域，而Object不能存储double值。

  • 运行时的类型查询只适用于原始类型。虚拟机中的对象总有一个特定的非泛型类型，所有的类型查询只产生原始类型。

  • 不能创建参数化类型的数组，需要时使用 ArrayList<Pair<String>> （安全有效）。

  • Varargs警告。向参数可变的方法传递一个泛型类型的实例，会有警告，可以使用 @SuppressWarnings("unchecked") 或 @SafeVarargs 标注该方法。

  • 不能实例化类型变量。不能使用new T(...), new T[...] 或 T.class这些表达式中的类型变量。可以通过反射调用Class.newInstance方法来构造泛型对象。

  • 泛型类的静态上下文中类型变量无效。

  • 不能抛出或捕获泛型类的实例

    • 既不能抛出也不能捕获泛型类对象；

    • 泛型类扩展Throwable也不合法。

    • catch子句不能使用类型变量

    • 在catch语句块内部是可以使用类型变量的

    • 可以利用泛型消除对已检查异常的检查（相当于将已检查异常包装成未检查异常）

  • 泛型规范一个原则：要想支持擦除后的转换，就需要强制一个类或类型变量不能同时成为两个接口类型的子类，而这两个接口是同一个接口的不同参数化。

  • 泛型类型的继承规则:Pair<S> 和 Pair<T>没有什么联系

  • 通配符类型：带有超类型限定的通配符可以向泛型对象写入，带有子类型限定的通配符可以从泛型对象读取。

    Pair<? extends Employee>   //子类型限定
Pair<? super Manager> // 超类型限定
