Selenium使用

selenium简单入门及注意点，内容多来自官网document的翻译

简介

selenium官方网站

Selenium automates browsers. That’s it! 简单来说就是使浏览器可以自动运行的工具，用于web应用程序ui的自动化测试比较多。

  1. Selenium 2.0 ≈ Selenium 1.x(Selenium-RC) + WebDriver
  2. Selenium-RC 浏览器启动后，采用注入js的方式驱动浏览器运行：对所有浏览器采用相同的方式。
  3. WebDriver 通过浏览器对自动化的原生支持，直接调用浏览器：不同浏览器有不同的特性，采用不同的驱动方式。

  4. Selenium目前工程有4个，分别为 Selenium WebDriver、Selenium Grid、Selenium IDE、Selenium Remote Control（Selenium RC）

定位元素

查找方式 描述 举例 (Java)
By.id 以元素的 id 属性来定位页面元素 By.id(“username”)
By.className 以元素的 class 属性来定位页面元素 By.className(“even-table-row”)
By.cssSelector 基于 CSS 选择器定位页面元素 By.cssSelector(“#username”)
By.TagName 以元素的标签名称定位页面元素 By.tagName(“td”)
By.name 以元素的 name 属性来定位页面元素 By.name(“username”)
By.LinkText 查找包含链接显示文案的页面元素 By.linkText(“Click Me!”)
By.xpath 以 XPath 来定位页面元素 By.xpath(“//*[@id=”username”])

显示、隐式等待（WebDriverWait）

页面元素有时加载较慢，无法立即获取到，需要等待一段时间，可以使用WebDriverWait，分显示和隐式两种方式。

注：官网建议不要混合使用显示和隐式这两种方式，否则等待时间可能比较奇怪。

显示等待

  • 等待10s超时，超时没有找到指定元素就抛出 TimeoutException
  • WebDriverWait默认 500ms 调用ExpectedConditions的方法一次，直到找到元素或超时。

隐式等待

  • 默认是等待0s
  • 一旦设置后，WebDriver的整个生命周期都有效
